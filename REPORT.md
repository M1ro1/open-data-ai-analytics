Звіт про виконання лабораторної роботи



1\. Мета роботи

Засвоєння практичних навичок роботи з системою контролю версій Git та платформою GitHub. Опанування процесу створення репозиторію, управління гілками (branching), злиття коду через Pull Requests та розв'язання конфліктів злиття на прикладі проєкту з аналізу даних.



2\. Контекст проєкту

В рамках лабораторної роботи було створено базову структуру проєкту для аналізу відкритих даних. 

&nbsp; Тема: Аналіз ринку транспортних засобів в Україні.

&nbsp; Джерело даних: Відкриті дані МВС щодо реєстрації транспортних засобів.

&nbsp; Основні завдання аналізу: Дослідження популярності електрокарів, вікової структури автопарку та колірних/брендових вподобань українців.



3\. Виконані кроки та Git Workflow

1\. Ініціалізація та налаштування:

&nbsp;  Створено репозиторій та налаштовано стандартну структуру директорій проєкту (data/, notebooks/, src/, reports/).

&nbsp;  Створено та налаштовано файл .gitignore для виключення з відстеження тимчасових файлів (кеш Python), віртуального середовища та великих файлів з даними.



2\. Ізольована розробка функціоналу (Feature Branching):

&nbsp;  Робота над кожним логічним етапом проєкту велася в окремих гілках:

&nbsp;    - feature/data\_load — додано скрипт для завантаження даних.

&nbsp;    - feature/data\_quality\_analysis — додано скрипт для перевірки якості даних.

&nbsp;    - feature/data\_research — додано скрипт для аналізу даних та оновлено документацію.

&nbsp;    - feature/visualization — додано код для побудови графіків.



3\. Код-рев'ю та злиття (Pull Requests):

&nbsp;  Усі зміни інтегрувалися в основну гілку main виключно через механізм Pull Requests на GitHub, що дозволило зберегти чисту історію комітів.



4\. Розв'язання конфліктів (Merge Conflicts):

&nbsp;  Під час паралельної розробки в гілках data\_quality\_analysis та data\_research виник конфлікт злиття у файлі README.md. Конфлікт було успішно локалізовано та вирішено вручну перед фінальним злиттям гілки дослідження в main.



5\. Реліз та версіонування:

&nbsp;    Після успішного злиття всіх компонентів було сформовано файл CHANGELOG.md з описом змін.

&nbsp;    На фінальний стан гілки main встановлено тег версії v0.1.0.
